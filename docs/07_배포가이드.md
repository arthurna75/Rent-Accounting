# 배포 가이드

## 1. Supabase 프로젝트 초기화

### 1-1. CLI 설치 및 초기화
```bash
npm install -g supabase

# 로그인
supabase login

# 프로젝트 초기화 (이미 존재하는 프로젝트에 연결)
supabase link --project-ref <your-project-ref>
```

### 1-2. Migration 실행
```bash
# 로컬 테스트
supabase db reset

# 프로덕션 적용
supabase db push

# 순서:
# 1. 20260618000001_initial_schema.sql
# 2. 20260618000002_rls_policies.sql
# 3. 20260618000003_seed_data.sql
```

### 1-3. 타입 생성
```bash
supabase gen types typescript --linked > src/types/supabase.ts
```

### 1-4. Supabase 대시보드 설정
1. **Authentication → Providers**: Email 활성화
2. **Authentication → Email Templates**: 한국어 커스터마이징
3. **Authentication → URL Configuration**:
   - Site URL: `https://your-domain.vercel.app`
   - Redirect URLs: `https://your-domain.vercel.app/auth/callback`
4. **Storage**: 계약서 파일 업로드용 버킷 생성 (선택)

## 2. Vercel 배포

### 2-1. 환경 변수 설정
Vercel 대시보드 → Settings → Environment Variables:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...  (서버 전용, NEXT_PUBLIC 제외)
NEXT_PUBLIC_APP_URL=https://your-domain.vercel.app
```

## 6. 회계연도 자동 생성 (pg_cron)

```sql
-- Supabase SQL Editor에서 실행
-- 매년 12월 31일 23:00에 다음연도 회계연도 자동 생성
select cron.schedule(
  'create-next-fiscal-year',
  '0 23 31 12 *',
  $$
    insert into public.fiscal_years (organization_id, year, start_date, end_date)
    select
      id,
      extract(year from now())::int + 1,
      date_trunc('year', now() + interval '1 year')::date,
      (date_trunc('year', now() + interval '2 years') - interval '1 day')::date
    from public.organizations
    where is_active = true
    on conflict (organization_id, year) do nothing;
  $$
);
```

## 7. 멀티테넌트 SaaS 운영

### 구독 플랜 관리
| 플랜 | 부동산 수 | 사용자 수 | 기능 |
|------|---------|---------|------|
| Basic | 5건 | 1명 | 기본 회계 |
| Pro | 30건 | 5명 | 전체 기능 |
| Enterprise | 무제한 | 무제한 | + API 접근 |

### 온보딩 플로우
1. `/auth/register` → 이메일 인증
2. `/auth/onboarding` → 조직(사업자) 정보 입력
3. 자동: Organizations 레코드 생성 + 기본 계정과목 seed
4. `/dashboard` → 대시보드 진입
